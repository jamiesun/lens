# 介绍

Lens 是一个**个人使用、本地优先**的浏览器前端 Agent Runtime，运行在 Chromium Manifest V3 扩展里。它通过 Side Panel 提供统一的对话式交互，通过结构化页面协议和一组受运行时严格管控的工具，理解用户自己的业务系统、协助填写表单、执行点击、截图，并且——这是本书的重点——可以直接调用业务前端自己注册的结构化工具。

```text
                           ┌────────────────────┐
                           │ Model Provider API │
                           └─────────▲──────────┘
                                     │ user-owned key
┌────────┐     ┌─────────────────────┴─────────────────────┐
│  User  │ ↔   │ Side Panel UI ↔ Local Agent Runtime       │
└────────┘     │                  ├─ Policy / Confirmation  │
               │                  ├─ Secret Vault           │
               │                  └─ Run State / Audit      │
               └─────────────────────┬─────────────────────┘
                                     │ typed messages
               ┌─────────────────────▼─────────────────────┐
               │ Content Adapter / Semantic Page Snapshot  │
               └─────────────────────┬─────────────────────┘
                                     │ validated bridge
               ┌─────────────────────▼─────────────────────┐
               │ Host Page Protocol：DOM 快照 + Page Tools  │
               └─────────────────────┬─────────────────────┘
                                     │
                           ┌─────────▼──────────┐
                           │ Existing Business  │
                           │ Frontend / Backend │
                           └────────────────────┘
```

模型只负责理解目标、提出结构化工具调用；权限、风险分级、确认、执行与记录全部由本地运行时裁决。模型服务使用用户自己的凭据直接连接，业务操作复用现有系统及其后端，Lens 不新增服务端、不做多租户、不托管密钥。

## 何时应该接入页面工具

如果你既维护 Lens，又维护它要操作的业务前端，通用的"读 DOM、猜测表单、模拟点击"策略会比直接调用业务前端暴露的结构化接口更慢、更脆弱。Lens Page Tools v1 就是为这种场景设计的私有协议：业务前端用几行 JavaScript 声明几个工具，Lens 运行时发现、校验并按风险分级决定是否放行执行，模型只在允许的范围内调用。

如果你的业务前端不受你控制（第三方网站），Lens 仍然可以用语义快照、表单填写和点击兜底工作，只是效率不如结构化工具。
