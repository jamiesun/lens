# Agent 工程约束

## 项目边界

- Lens 是个人使用、本地优先的浏览器前端 Agent Runtime，不是通用自主浏览器代理。
- 除非用户明确修改项目边界，否则不得引入新的 Agent 服务端、多租户、计费、团队权限或集中式密钥托管系统。
- 模型只能提出结构化工具调用；权限、风险、确认、审计和执行结果必须由受控运行时裁决。
- 不得提供任意 JavaScript、任意网络请求或默认覆盖所有网站的万能能力。

## 验收矩阵（硬性规定）

以下规则均为 MUST 级要求，完整矩阵维护在 [`docs/roadmap.md`](docs/roadmap.md#验收矩阵业务能力覆盖矩阵)：

1. 每个一级功能必须至少有一条 Happy Path E2E 验证。
2. 每个高风险功能必须至少覆盖一条失败路径。
3. 每个涉及权限的功能必须至少验证两种角色或访问状态。
4. 每个会修改系统状态的操作必须至少验证一次失败后的恢复或回滚。
5. 每次新增一级业务功能，必须同步新增对应的 E2E 并更新 `docs/roadmap.md` 的验收矩阵，否则变更不完整。
